## プロジェクト概要

ai_improve_issue は簡易なブランクIssueに1-2行程度のやりたいことを書いたら、適切なテンプレートに沿った具体的な文章を自動生成し、コメントで提示する機能です。

## 開発ガイドライン

- 全てのコミュニケーション（PR本文、PRコメント、ソースコメント、docstring等）は日本語で書いてください。
- コードから明らかな情報をソースコメントに書かないでください。
- コード修正後は `uv format` コマンドで整形してください

## 要件、設計、実装

- 要件は `/.dev/requirements/` 以下に `issue番号_要件_要件名.md` であります。
- 設計は `/.dev/designs/` 以下に `issue番号_設計_設計名.md` であります。
- 実装は `/.dev/implementations/` 以下に `issue番号_実装_実装名.md` であります。

## 実装内容の共有

- 現在のGitブランチのdiffなどを元に、対応した作業内容をまとめてください。
- 対応した内容は、「要件」「設計」「実装」に応じて前述の各ディレクトリにファイルを追加してください。
- GitHub Pull Request では、実装内容をPull Request 本文に日本語で、毎回更新してください。
- GitHub Pull Request では、タイトルを日本語にし、先頭に `[#issue番号]` を付けてください。

実装内容の出力形式

```
## 対応内容

対応範囲

- xxxxxx
- xxxxxx

対応範囲外

- （あれば、記載）

## 技術的な注意点、制限事項

- xxxxxx
- xxxxxx

## 確認してほしいこと

- xxxxxx
- xxxxxx

```

## 基本方針

- 実装時は既存コードの設計パターンに従ってください。
